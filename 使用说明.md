# 🌰 榴莲庄经营追踪系统 · 使用说明（新手版）

这套系统帮你解决一个核心痛点：**榴莲损耗分散在各环节（园区坏果、运输坏果、商家拒收），导致真实成本和利润算不准。**
系统用「真实可售量」算成本，把坏掉的果换算成真实亏的钱，让你看清**到底赚没赚、钱漏在哪、怎么提业绩**。

你得到两样东西：
1. **一个网页仪表盘**（记录 + 自动算账 + 图表）
2. **一支 5 人 AI 团队**（在 Claude Code 里随叫随到）

---

## 一、打开仪表盘（每天用这个记账看数）

1. 打开 **Google Chrome 浏览器**（推荐 Chrome，能一键连数据文件；Safari 也能用但要手动导入导出）。
2. 把这个文件拖进 Chrome，或在地址栏打开：
   `dashboard/index.html`
3. 你会看到示例数据（猫山王、黑刺两批），先随便点点熟悉一下。

### 让 AI 团队能读到你的数据（重要）
- 点右上角 **「📂 连接数据文件」** → 选中 `data/farm-data.json` 文件。
- 连上后，**你之后每次修改都会自动写回这个文件**，AI 团队就能读到最新数据。
- 每次重新打开仪表盘后，想让 AI 看最新数据，**记得再点一次连接**。
- 不用 Chrome 时：用「导出 JSON」备份，用「导入 JSON」恢复。

### 六个页面
| 页面 | 干什么 |
|---|---|
| 📊 总览 Overview | 收入、真实成本、净利、净利率、损耗率、**A 级占比** + 图表 + 经营预警 |
| 🌳 采收批次 Batches | 每次采收记一批：品种、数量、园区坏果、**A/B/C 分级**、树龄、采收方式、各项成本 |
| 💵 成本 Costs | 固定成本（土地租金/折旧）+ 各批变动成本一览 |
| 🚚 批发与损耗 | 记批发订单（含**等级**、运输坏果、商家拒收、实收、回款）+ 损耗分析；卖价低于目标价会**红字提醒** |
| 🏅 等级与定价 Grades | 等级构成、各级成交价 vs 目标价、**目标价对照表**、升级空间（多赚多少）|
| 🧾 应收/客户 | 按商家看销量、回款、**谁还欠钱** |

> 🏅 **关于等级**：一棵树掉下来的果是 A/B/C 混在一起的。**成本是整批共享的（同树同肥），等级只影响收入**——所以 A 级占比越高、越接近目标价，利润越好。这是提利润最大的杠杆。采收时顺手把好果分成 A/B/C 各多少公斤填进去即可。

> 💡 **不确定的数字先填 0**，以后补。先把流程跑起来比一开始就完美更重要。

### 备份
- 随时点 **「⬇ 导出 Excel」** 下载一份 CSV（可用 Excel/手机打开，也能发给员工或商家看）。
- 或让 **行政 AI** 帮你定期备份。

## 二、你的 5 人 AI 团队（在 Claude Code 里用）

在这个项目目录下打开 Claude Code，直接用大白话叫他们就行。他们都会先读 `data/farm-data.json` 再回答，用中文。

| 同事 | 找他做什么 | 例子怎么说 |
|---|---|---|
| **PA 农场助理** (`durian-pa`) | 日常总入口、帮你记数、出每日/每周汇报、不知道找谁就找他 | 「今天 A 区采了 320 颗约 640 公斤，坏了 35 公斤，人工花了 800，帮我记一下」 / 「这周经营怎么样？」 |
| **行政** (`durian-admin`) | 检查数据有没有记错、加商家档案、备份 | 「帮我检查数据有没有错」 / 「帮我备份」 / 「加个新商家：城北水果行」 |
| **会计** (`durian-accountant`) | 算真实成本、每批盈亏、损耗核算、**各等级收入**、应收、月结 | 「上个月哪批其实是亏的？」 / 「我的 A/B/C 级各卖了多少、均价多少？」 / 「帮我做6月月结」 |
| **财务/CFO** (`durian-finance`) | 我到底赚没赚、怎么提利润、**等级/定价**、现金流、风险 | 「提升 A 级占比能帮我多赚多少？」 / 「有哪些单卖得低于目标价？」 / 「运输损耗能帮我省多少钱？」 |
| **营销+内容** (`durian-marketing`) | 找批发商、写招商话术/**分级报价单**、做小红书/抖音/FB 内容 | 「帮我做一份按 A/B/C 分级的报价单」 / 「做一条猫山王老树果的小红书」 / 「A 级凭什么卖更贵，帮我写话术」 |

**怎么叫：** 直接说需求即可，例如
> 「帮我问会计：这两批榴莲到底哪批赚钱？」

不确定找谁时，就找 **PA**，他会帮你分流。

## 三、系统怎么算账（灵魂公式）

```
好果(A+B+C) = 采收量 − 园区坏果
真实可售量 = 好果 − 运输坏果 − 商家拒收
真实单位成本 = 真实总成本 ÷ 真实可售量      ← 分母是「能卖的量」，不是「采的量」
某环节损耗金额 = 该环节损耗量 × 真实单位成本   ← 坏掉的果 = 真实亏的钱
净利 = 实收收入 − 真实总成本
```

> ⚠️ 很多新手用「采收量」算成本，所以**单位成本算低了、利润算高了**。
> 本系统用「真实可售量」，算出来才是你**真实的利润**。
> 固定成本（土地/折旧）会按当月各批采收重量自动分摊到每批。
> 🏅 **等级不进成本、只进收入**：同一批果成本共享，A 级卖得贵 → A 级占比和卖价决定这批利润厚不厚。

## 四、建议的日常节奏
- **每次采收**：在「采收批次」记一批（品种、重量、园区坏果、**A/B/C 各多少公斤**、树龄、采收方式、当批成本）。
- **每次发货给商家**：在「批发与损耗」记一单（**选等级**、运输坏果、商家拒收、实收、回款状态）。卖价低于目标价会红字提醒。
- **每周**：找 **PA** 要一份周报；找 **会计** 看损耗和应收；找 **财务** 看**怎么提 A 级占比/卖到目标价**。
- **每月**：找 **会计** 做月结，找 **财务** 做下月计划，找 **营销** 安排拓客/内容。
- **每周或大改前**：让 **行政** 帮你备份。

## 五、备份 + 换电脑（云端同步）

你的数据已经接到 GitHub 云仓库：`https://github.com/yongyi93-code/duriansystem`
文件夹里有两个**双击就能跑**的脚本，平时只用记住两句话：

| 脚本 | 什么时候双击 | 它做什么 |
|---|---|---|
| 🟢 **同步下载.command** | **开工前** / 换电脑后 | 从云端把最新数据拿下来 |
| 🔴 **备份上传.command** | **收工后** / 记完账 | 把这台电脑的改动存到云端 |

> 它不是云盘、不会自动同步——**得你自己双击**。习惯就是：开工先「同步下载」，收工再「备份上传」。
> 第一次双击若提示「身份不明的开发者」：**右键点脚本 →「打开」→ 再点「打开」**，以后就能直接双击。
> 脚本有防呆：没改动会告诉你「不用上传」；本地有没存的改动时同步会先警告，不会偷偷覆盖。

**换一台新电脑**（那台还没有这个文件夹）：先在「终端 Terminal」里跑一次
```
git clone https://github.com/yongyi93-code/duriansystem.git
```
下载好后，那台电脑也有这两个脚本，往后照样双击即可。
